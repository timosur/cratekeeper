## How to use it

### Option A — Incremental (recommended)

Implement one milestone at a time. Best when you want to ship a usable shell quickly, then layer features.

1. Copy this `frontend-plan/` folder into your implementation codebase.
2. Open [prompts/section-prompt.md](prompts/section-prompt.md), replace the `SECTION_NAME`, `SECTION_ID`, and `NN` placeholders for the milestone you're working on, and add any notes in the Notes section.
3. Paste the prompt into your coding agent (Claude Code, Cursor, Copilot, …).
4. Answer the agent's clarifying questions (tech stack, what's already in place, backend wiring).
5. Let the agent implement based on the per-milestone instructions in [instructions/incremental/](instructions/incremental/).
6. Repeat for each milestone:
   1. **01-shell** — design tokens + the persistent left-sidebar shell
   2. **02-events** — dashboard card grid (the entry point)
   3. **03-event-detail** — the 11-step pipeline working surface
   4. **04-master-library** — curated collection on local OS disk
   5. **05-settings** — integrations, FS roots, genre buckets, API access
   6. **06-audit-log** — append-only filterable timeline

### Option B — One-shot

Implement everything in a single session. Best for a greenfield prototype with a strong agent and clear requirements.

1. Copy this `frontend-plan/` folder into your implementation codebase.
2. Open [prompts/one-shot-prompt.md](prompts/one-shot-prompt.md), add any notes in the Notes section.
3. Paste the prompt into your coding agent.
4. Answer its clarifying questions.
5. Let the agent implement using [instructions/one-shot-instructions.md](instructions/one-shot-instructions.md).

## Testing

Each section ships with a `tests.md` file in [sections/<section-id>/](sections/) describing UI behavior tests in framework-agnostic terms. After implementation, you should be able to translate them directly into your test runner of choice (Vitest + Testing Library, Playwright, etc).

## Tips

- **Run the agent's clarifying questions to completion.** The prompts are designed to surface tech-stack and integration questions early; resist the urge to skip them.
- **Components are props-based.** All callbacks are optional. Render with `sample-data.json` first, then wire to real APIs.
- **Sky / emerald / neutral / amber / red** are the only colors. No custom hexes.
- **Tailwind v4 only.** No `tailwind.config.js`.
- **Inter and JetBrains Mono** are the only typefaces. Identifiers and counts are always mono.
- **Dark mode** is mandatory across every surface.
